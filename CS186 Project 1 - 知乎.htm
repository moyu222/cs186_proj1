Question 1

Question 1的四个小问比较简单，全部是单表查询。其中，第二个小问涉及到正则表达式，后面小问涉及到分组查询。

第一小问实现代码：

CREATE VIEW q1i(namefirst, namelast, birthyear)
AS
  SELECT namefirst, namelast, birthyear
  FROM people
  WHERE weight > 300
;

第二小问实现代码：

CREATE VIEW q1ii(namefirst, namelast, birthyear)
AS
  SELECT namefirst, namelast, birthyear
  FROM people
  WHERE namefirst REGEXP "\s"
;

第三小问实现代码：

CREATE VIEW q1iii(birthyear, avgheight, count)
AS
  SELECT birthyear, AVG(height), COUNT(*)
  from people
  group by birthyear
  order by birthyear asc
;

第四小问实现代码：

CREATE VIEW q1iv(birthyear, avgheight, count)
AS
  SELECT birthyear, AVG(height), COUNT(*)
  from people
  group by birthyear
  having AVG(height) > 70
  order by birthyear asc
;

Question 2

Question 2的引入的知识点是连接查询，要注意的在HallofFame表中存在字段inducted标注是否入选名人堂。

由于不熟悉SQL语言，第二小问是一个小难题，球员信息、名人堂信息和学校信息分散在三张表中，需要连续进行内连接或者左连接。

第一小问实现代码：

CREATE VIEW q2i(namefirst, namelast, playerid, yearid)
AS
  select p.namefirst, p.namelast, p.playerid, h.yearid
  FROM halloffame AS h
  LEFT JOIN people AS p ON p.playerid = h.playerid
  where h.inducted = "Y"
  order by h.yearid DESC, p.playerid asc
;

第二小问实现代码：

CREATE VIEW q2ii(namefirst, namelast, playerid, schoolid, yearid)
AS
  SELECT p.namefirst, p.namelast, p.playerid, s.schoolID, h.yearid
  FROM halloffame AS h
  LEFT JOIN people AS p ON p.playerid = h.playerid
  INNER JOIN collegeplaying AS c ON h.playerid = c.playerid
  INNER JOIN schools AS s ON c.schoolid = s.schoolID
  WHERE s.schoolState = "CA" and h.inducted = "Y"
  ORDER BY h.yearid DESC, c.schoolid ASC
;

第三小问实现代码：

CREATE VIEW q2iii(playerid, namefirst, namelast, schoolid)
AS
  SELECT p.playerid, p.namefirst, p.namelast, c.schoolID
  FROM halloffame  as h
  LEFT JOIN collegeplaying AS c ON h.playerid = c.playerid
  INNER JOIN people AS p on p.playerid = h.playerid
  where h.inducted = "Y"
  order by h.playerid DESC, c.schoolID asc
;

Question 3

Question 3引入了计算、having关键字和嵌套表达式。前两问比较常规，在第三问中需要先利用嵌套表达式计算出指定运动员的lslg值，然后再找出所有lslg值大于他的运动员。

第一小问实现代码：

CREATE VIEW q3i(playerid, namefirst, namelast, yearid, slg)
AS
  SELECT b.playerid, p.namefirst, p.namelast, b.yearid, (b.h + b.h2b + 2.0*b.h3b + 3.0*b.hr) / b.ab AS slg
  FROM batting AS b
  INNER JOIN people p on p.playerid = b.playerid
  WHERE b.AB > 50
  ORDER BY slg DESC
  limit 10
;

第二小问实现代码：

CREATE VIEW q3ii(playerid, namefirst, namelast, lslg)
AS
  SELECT b.playerid, p.namefirst, p.namelast, (SUM(b.h) + SUM(b.h2b) + SUM(2.0*b.h3b) + sum(3.0*b.hr)) / SUM(b.ab) AS lslg
  FROM batting AS b
  INNER JOIN people p on p.playerid = b.playerid
  GROUP BY b.playerid
  HAVING SUM(b.ab) > 50
  ORDER by lslg DESC
  limit 10
;

第三小问实现代码：

CREATE VIEW q3iii(namefirst, namelast, lslg)
AS
  SELECT p.namefirst, p.namelast, (SUM(b.h) + SUM(b.h2b) + SUM(2.0*b.h3b) + sum(3.0*b.hr)) / SUM(b.ab) AS lslg
  FROM batting AS b
  INNER JOIN people p on p.playerid = b.playerid
  GROUP BY b.playerid
  HAVING SUM(b.ab) > 50
  AND lslg > (
    select (SUM(b.h) + SUM(b.h2b) + SUM(2.0*b.h3b) + sum(3.0*b.hr)) / SUM(b.ab)
    from batting AS b
    GROUP BY b.playerid
    having b.playerid = "mayswi01"
  )
  ORDER by lslg DESC
;

Question 4

在Question 4中，大量使用了公共公共表表达式和一些小技巧。

第二小问是个均匀分箱任务，首先计算每个箱子的区间长度，然后利用整数除法得到每条记录归属的箱号，具体公式是箱号=MIN(CAST((记录 - 最小值)/区间宽度 AS INT), 9)，值得注意的是，这里使用了CAST关键字将箱号映射到整数上，使用MIN将最大值的箱号映射到9而非计算结果10上面。然后根据计算得到的每条记录归属的binid分组即可实现分箱。

第三小问是求逐差，这里有个技巧是将同一张表进行错位自连接，具体来说，就是按照年份连接，且一条记录里左边的表年份比右边的小1，这样直接对同一条记录的两个数据作差即可得到逐差。

第一小问实现代码：

CREATE VIEW q4i(yearid, min, max, avg)
AS
  SELECT yearid, min(salary), max(salary), avg(salary)
  from salaries as s
  group by yearid
;

第二小问实现代码：

CREATE VIEW q4ii(binid, low, high, count)
AS
  with salaries2016 as(
    select * from salaries
    where yearid = 2016
  ),
  salaryStats as (
    select MIN(salary) as mins, MAX(salary) as maxs, (MAX(salary)-MIN(salary))/10.0 as width
    from salaries2016
  ), salaryBins as (
    select s.salary, MIN(CAST((s.salary - salaryStats.mins)/width AS INT), 9) AS binid
    from salaries2016 s, salaryStats
  )
  SELECT id.binid, id.binid*(select width from salaryStats)+(select mins from salaryStats), (id.binid+1)*(select width from salaryStats)+(select mins from salaryStats), count(*)
  from binids id LEFT JOIN salaryBins sb on id.binid = sb.binid
  group by id.binid
;

第三小问实现代码：

CREATE VIEW q4iii(yearid, mindiff, maxdiff, avgdiff)
AS
  with yearlyData AS(
    select yearid, min(salary) as mins, max(salary) as maxs , avg(salary) as avgs
    from salaries
    group by yearid
  )
  select d2.yearid, d2.mins-d1.mins, d2.maxs-d1.maxs, d2.avgs-d1.avgs
    from yearlyData d1
    INNER JOIN yearlyData d2 on d1.yearid+1 = d2.yearid
    order by d1.yearid asc
;

第四小问实现代码：

CREATE VIEW q4iv(playerid, namefirst, namelast, salary, yearid)
AS
  with maxSalaries as (
    select yearid, max(salary) as salary
    from salaries
    where yearid = 2001 or yearid = 2000
    group by yearid
  )
  select p.playerid, p.namefirst, p.namelast, ms.salary, ms.yearid
  from maxSalaries ms
  JOIN salaries s
  on ms.yearid = s.yearid and ms.salary = s.salary
  JOIN people p
  on s.playerid = p.playerid

  order by ms.yearid asc
;

第五小问实现代码：

CREATE VIEW q4v(team, diffAvg) AS
  with all2016 as (
    select * from allstarfull
    where yearid = 2016
  )
  select s.teamid, max(s.salary) - min(s.salary)
  from all2016
  JOIN salaries s
  ON all2016.playerid = s.playerid and all2016.teamid = s.teamid and s.yearid = 2016
  group by s.teamid
;
